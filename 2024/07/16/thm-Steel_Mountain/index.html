thm靶场wp-Steel Mountain

扫描一下端口拿一下信息

看了一下 80 端口放了一张员工照片

8080端口是一个HttpFileServer服务

是 Rejetto Http File Server 服务

查询CVE，比较匹配的是 CVE-2014-6287

进msf搜一下

正好匹配

查看一下需要设置的内容

查看一下由于服务在8080端口，所以需要更改RPORT，设置一下RHOSTS即可

成功拿到shell

查看发现是Windows系统

加载Windows的powershell

https://raw.githubusercontent.com/PowerShellMafia/PowerSploit/master/Privesc/PowerUp.ps1

用此脚本来进行评估 Windows 机器并确定任何异常情况

按ctrl + z 切换到 meterpreter 命令行

用upload命令上传下载的脚本

通过命令运行

得到了一堆服务

通过分析，可以把关注点放在AdvancedSystemCareService9

Check显示是没有加引号的服务路径

也就是C:\Program Files (x86)\IObit\Advanced SystemCare\ASCService.exe中的Advanced SystemCare路径可以被利用

解释：

现在利用msf生成一个shell，命名需要为Advanced.exe

在本地监听4443端口

用upload命令上传shell

将shell移动到C:\Program Files (x86)\IObit\目录中，这样就可以替代原本的服务路径执行我们的shell了

现在重启一下服务即可

如果是cmd命令行，sc.exe需要换成sc
